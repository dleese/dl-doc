= eForms
The _eforms_ Module in Logipad provides pilots with the ability to manage and access electronic forms, checklists, and other operational documents. It serves as a digital platform for storing, organizing, and completing various types of forms and checklists related to flight operations. _eforms_ can be created with Logipad's eFormGenerator tool, which allows users to design and customize forms according to their specific needs and requirements.

the following list are some examples of possible eForms:

* Flight Crew Report
* Technical Log
* Cabin Log
* Cabin Safety Report
* Occurrence Report
* Flight Safety Report
* Voyage Report
* Feedback Report
* Maintenance Inspection
* Discretion Report

eForms are not related to Pilots only. The eForms module can be used by all crew members, including cabin crew and ground staff. The module allows users to create, edit, and submit forms electronically, ensuring that all necessary information is captured accurately and efficiently.

After an _eForm_ is completed, it can be submitted electronically to the Logipad server for archiving and analysis. Workflows can also be defined to ensure that the completed forms are routed to the appropriate personnel for review and action. eForms data will be stored in LOgipad's internal database and can be accessed by authorized users for reporting and analysis purposes. The data can also be exported in various formats (e.g., CSV, PDF) for further processing or integration with other systems.
The _eForms_ module also supports offline access, allowing users to complete forms even without an internet connection. Once connectivity is restored, the completed forms will be synchronized with the Logipad server. 

== eForms Features
The _eforms_ module includes the following features to assist pilots in managing and accessing forms effectively: 

* Fields can be marked as mandatory or optional 
* Fields can be pre- validated with a specific format (e.g., date, time, number)
* Signatures can be added to the _eforms_
* Pictures can be added to the _eforms_ ( e.g., for damage reports)
* eForms can be submitted electronically to the Logipad server
* eForms can be saved as drafts for later completion

== eForm states
The _eForms_ module includes different states for eForms to help users manage their forms effectively. The following states are available:

* *_Draft_* eForms: eForms that have been started but not yet completed. Drafts can be saved and edited later.
* *_Outbox_* eForms: eForms that have been submitted but not yet uploaded to the Logipad server. The outbox will be stored locally on the device until it is uploaded to the Logipad server.
* *_Sent_* eForms: eForms that have been successfully uploaded to the Logipad server.

== eForm Drafts
An eForm Draft is an eForm that has been started but not yet completed. The draft can be saved and edited later. The draft will be stored locally on the device until it is submitted to the Logipad server. A Draft can also be used as a template for future eForms. For example, if a user frequently fills out the same form, they can create a draft and use it as a template for future submissions. This saves time and ensures consistency in the data entered.
Users can create multiple drafts of the same eForm. Drafts can be manually deleted by the user with a left swipe gesture on the draft cell in the eForm view. Possible actions for the eForms in this section are:

* *_Export as PDF to Transfer_* Export the eForm to a PDF file and copy it to the Logipad _Transfer_ section. 
* *_Export as PDF per Email_* Export the eForm to a PDF file and send it via email. The email will be sent from the default email client on the device. The user can select the recipient and add additional information to the email.
* *_Print_* Print the eForm
* *_Delete_* Delete the eForm from the device. The eForm will be deleted from the device and cannot be recovered. The eForm will be removed from the _eForm Sent_ section.

== eForm Outbox
eForms which are visible inside the _eForm Outbox_ section has been submitted but not yet uploaded to the Logipad server. The reason could be a WiFi outage or no availability of a WiFi connection. The content of the outbox will be stored locally on the device until it is uploaded to the Logipad server. Once the device is connected to the internet, the content of the outbox will be automatically uploaded to the Logipad server. The eForm section is cleared and content is moving from the outbox to the _eForm Sent ( read only)_ section.
Possible actions for the eForms in this section are:

* *_Export as PDF to Transfer_* Export the eForm to a PDF file and copy it to the Logipad _Transfer_ section. 
* *_Export as PDF per Email_* Export the eForm to a PDF file and send it via email. The email will be sent from the default email client on the device. The user can select the recipient and add additional information to the email.
* *_Print_* Print the eForm
* *_Delete_* Delete the eForm from the device. The eForm will be deleted from the device and cannot be recovered. The eForm will be removed from the _eForm Sent_ section.

== eForm Sent
eForms which have been sent to the Logipad System are visible inside the _eForm Sent_ section. The eForms in this section are read only and cannot be edited. The eForms in this section can be exported to a PDF file or printed. The eForms in this section can also be deleted by the user with a left swipe gesture on the eForm cell in the eForm view. Possible actions for the eForms in this section ar

* *_Export as PDF to Transfer_* Export the eForm to a PDF file and copy it to the Logipad _Transfer_ section. 
* *_Export as PDF per Email_* Export the eForm to a PDF file and send it via email. The email will be sent from the default email client on the device. The user can select the recipient and add additional information to the email.
* *_Print_* Print the eForm
* *_Delete_* Delete the eForm from the device. The eForm will be deleted from the device and cannot be recovered. The eForm will be removed from the _eForm Sent_ section.
* *_Copy_* Copy the eForm to the _eForm Draft_ section. The eForm will be copied to the _eForm Draft_ section with the same content as the original filled eForm. The eForm will moved to the _eForm Draft_ section. The eForm can be edited and submitted again.
