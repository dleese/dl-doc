= Briefing Module

The Briefing module provides access to flight-related information (for example the operational flight plan, weather, and NOTAMs). Crew members can view and download briefing packages to prepare for a flight, including for offline use when connectivity is limited.

Logipad offers a comprehensive briefing solution that includes the following features:

* Logipad is independent from the flight planning system and can be used with any flight planning tool.
* Logipad supports the ARINC 633-3 format, which is the standard for flight planning and briefing data exchange.
* Access to flight plans, weather reports, and NOTAMs
* Customizable briefing packages based on user preferences
* Support for multiple aircraft types and configurations
* Offline access to briefing information
* User-friendly interface for easy navigation and information retrieval
* Integration with other Logipad modules for seamless workflow

== Briefing Module Overview
The Briefing list shows all briefings assigned to the user. The list can be filtered (for example by date, flight number, or aircraft type). Each briefing provides access to package content such as weather, NOTAMs, and the flight plan.

Briefings can be downloaded for offline use. Before recording flight-related data, the relevant briefing must be activated. After flight completion, the briefing can be finalized and uploaded to the Logipad server for archiving (retention period is defined by the operator; a typical configuration is 90 days).

The Briefing module allows crew to activate a briefing before flight. A briefing can be activated from the briefing list or from within the briefing package itself.

Depending on operator configuration, activation may be available to both pilots. Finalization/sign-off is typically restricted to the Pilot in Command (Master User).

Do not use a workflow that bypasses required approvals or sign-off steps.

== Global Briefing Settings
Global Briefing Settings are configured by the Logipad administrator and apply to all users and briefing packages.

[.center-caption]
.Global Briefing Settings View
image::ios_global_briefing_settings.png[Global Briefing Settings view,75%,float="center",align="center"]

.Global Briefing Settings
* *Recent Briefings Filter*
    ** _none_ Show all briefings
    ** _STD_ Not later than 24 hours before the scheduled time of departure

* *Remove archived briefings after*
    ** _2 weeks_ Remove archived briefings after 2 weeks
    ** _1 month_ Remove archived briefings after 1 month
    ** _3 months_ Remove archived briefings after 3 months
    ** _6 months_ Remove archived briefings after 6 months
    ** _1 year_ Remove archived briefings after 1 year
    ** _Never_ Never remove archived briefings

* *Display F/U-Updates separately*
    ** _yes_ Show F/U-Updates in the briefing detailed view
    ** _no_ Show latest package only. The latest version can also be a merged updated package with the latest full updated version.

* *Upload finalized briefings immediately*
    ** _yes_ Upload finalized briefings to the Logipad server when connectivity is available.
    ** _no_ Upload is manual. The user uploads by pull-to-refresh (swipe down) or by tapping *Synchronize* on the Home view.

* *Automatic Briefing Transfer*
    ** _yes_ Recorded data is transferred automatically to other devices working on the same Flight ID. This helps the Master User and First Officer stay synchronized. Requirement: the other devices must also have this setting enabled.
    ** _no_ Recorded briefing data is not transferred automatically. The user transfers data manually (see Manual briefing transfer).

== Briefing Activation
Tap on the three dots on the right hand side of the briefing package inside the main briefing view. Depending on the user role _Master User_ or _First Officer_, the options are different. See below the options for the _Master User_ and _First Officer_.

NOTE: If automatic briefing transfer is enabled, activating a briefing may also activate it on other devices using the same flight (Flight ID), depending on configuration.

[.center-caption]
.Briefing with Master User assigned
image::ios_Briefing_MasterUser_HKG_CVG_Activate_Master.png[Briefing activation menu — Master User assigned,75%,float="center",align="center"]

While the Master User is shown *activate* only, the First Officer is shown *activate non‑Master copy* and (if enabled) *Force failover*. See screenshot below.

[.center-caption]
.Briefing with no Master User assigned
image::ios_Briefing_NoMasterUser_HKG_CVG_Activate_Master.png[Briefing activation menu — no Master User assigned,75%,float="center",align="center"]

* *activate non‑Master copy* indicates that the current user is assigned as First Officer for that flight. First Officers typically cannot finalize briefing packages.

* *Force failover* should only be used when the Master User cannot sign/finalize the briefing (for example due to device failure). It transfers Master responsibilities to the First Officer so the briefing can be completed.

[WARNING]
====
Force failover responsibility::
Use _Force failover_ only when the Master User cannot finalize the briefing (for example due to device failure). Ensure the First Officer is fully aware of the implications of taking over the Master User role.
====

When a briefing is activated, its status changes to *active* and the icon on the left turns green. When you open the briefing package, the status is shown in the top-left corner. If automatic briefing transfer is enabled, a blinking green dot may be shown.

== Briefing Menu Content
When opening a Briefing package, the briefing content can look similar to the screenshot below. On the left-hand side, you see sections of the current content structure of the briefing. The content and structure are individually configured for each airline and can differ. Keep in mind that the content structure is different for each airline and can be customized. The example below is just a sample of the content structure.

[.center-caption]
.Briefing Menu Content Example
image::ios_Briefing_MasterUser_HKG_CVG_Open_Master.png[Logipad briefing package content structure,75%,float="center",align="center"]

The sections typically include:

* *Flight Plan* — Detailed operational flight plan with routing and fuel calculations.
* *Weather* — Weather reports and forecasts for departure, en-route, and arrival airports.
* *NOTAMs* — Notices to Airmen relevant to the flight.
* *Weather Charts* — Graphical weather representations, such as wind charts or turbulence maps.
* *Additional Documents* — Any supplementary documents required for the flight, such as airport charts or airline-specific instructions.

Each section can be expanded to view detailed information, and users can navigate between sections using the menu on the left-hand side.
